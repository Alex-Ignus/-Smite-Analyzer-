Project Overview


Smite is a video game, more specifically a multiplayer online battle arena or MOBA for short. It's a game where players queue for different types of game-modes where the objective is to take down the enemies defenses and ultimately slay their titan.

My goal is to utilize gameplay data from the Smite developer API to design a better build analyzer that new and old players alike can use in order to see the viability of their build/strategy from their gameplay data.    

Features that are complete

Completed the backend ETL script that pulls data from Smite's API, performs the necessary backend processing, and uploads the data to the google Firestore database.  

 Upcoming tasks/Milestones,

I've figured out different ways to store the data in firebase so that it's more usable to me, and after receiving feed back on what kind of visualizations I should focus on, I plan on changing my first and second visualzations into less complex designs 

Challenges and Blockers

My main blocker as of late was getting data from firestore in a meaningful way so that I can visualize that data. one of the main problems here comes from the fact that the firestore passes over an array instead of an object to js from the firestore container. This isn't a problem when dealing with top level data, but if you have a subcontainer (a container inside a container) that you need to access, then you run into the issue of any data you retrieve when is considered undefined outside of the scope of the loop you used to get the data. 

The issue of accessing subcollections affects my second visualization the most as it releases on 2 different subcollections of 2 collections.
